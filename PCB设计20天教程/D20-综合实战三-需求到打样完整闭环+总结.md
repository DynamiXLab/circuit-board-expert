## 📖 第三部分：进阶方向

20 天学完，你已经是能独立做板的"入门级硬件工程师"了。想继续深入的话：

### 路线 A：嵌入式系统设计

```
学习内容：
├── 单片机选型（STM32 / ESP32 / Raspberry Pi Pico）
├── 传感器接口（I²C、SPI、UART、ADC）
├── 电源设计（DC-DC、LDO、电池管理）
├── 四层板设计（信号完整性入门）
└── 高速/射频设计（DDR、USB 3.0、天线匹配）
```

建议项目：STM32 温湿度计 / ESP32 WiFi 智能开关

### 路线 B：电源设计

```
学习内容：
├── Buck/Boost 变换器原理
├── LDO 选型与热设计
├── 电源完整性与去耦网络
├── 大电流布局技巧
└── 开关电源噪声抑制
```

建议项目：USB 可调电源 / 锂电池充电板

### 路线 C：模拟电路

```
学习内容：
├── 运放应用（滤波、放大、比较器）
├── ADC 采样前端设计
├── 低噪声布局技术
├── 传感器信号调理
└── 模拟与数字地分割（AGND / DGND）
```

建议项目：音频放大器 / 毫欧计 / 光电池检测

## 📖 最后的建议

> **"硬件设计不是看会的，是做会的。"**

1. **不要等"学会了再动手"**——从今天就可以开始你的下一个项目
2. **每块板子至少打样一次**——看实物和看截图完全是两回事
3. **保留迭代记录**——V1.0 有什么问题，V1.1 改了什么，记下来
4. **看别人的板子**——拆开手机、充电器、开发板，看别人怎么布局的
5. **出错是常态**——第一块板子就有 bug 是正常的，关键是能定位和修复
