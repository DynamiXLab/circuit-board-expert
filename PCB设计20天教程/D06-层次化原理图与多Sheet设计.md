# D6 — 层次化原理图与多 Sheet 设计

> **学习目标**：当原理图变复杂时，学会用多张 Sheet 组织设计，让原理图清晰、模块化、易维护。
>
> 📌 本教程基于 **嘉立创 EDA 专业版 v3.2.131**。

---

## 📖 为什么需要多 Sheet？

当原理图复杂度上升（几十上百个元件），一张图全画出来会变成"蜘蛛网"。

**层次化方法：**

- **根 Sheet（Root Sheet）** — 用方块图表示各功能模块的互连关系
- **子 Sheet（Child Sheet）** — 每个方块对应一张子原理图，画具体电路

### 典型模块划分示例

```
┌────────────────────────────────────────────────────┐
│                 Root Sheet                          │
│                                                    │
│  ┌────────────┐  ┌────────────┐  ┌──────────────┐ │
│  │  电源模块   │──│  MCU 模块   │──│  输出/驱动模块 │ │
│  │  (Sheet1)  │  │  (Sheet2)  │  │  (Sheet3)    │ │
│  └────────────┘  └────────────┘  └──────────────┘ │
│       │               │                │           │
│  ┌────────────┐  ┌────────────┐  ┌──────────────┐ │
│  │ 5V→3.3V   │  │  STM32     │  │  OLED 显示   │ │
│  │ 稳压输出   │  │  最小系统  │  │  驱动电路    │ │
│  └────────────┘  └────────────┘  └──────────────┘ │
└────────────────────────────────────────────────────┘
```

## 🔑 嘉立创 EDA 层次化操作步骤

### 第一步：创建子 Sheet

工程面板 → 右键 → ”新建板子“ → "新建原理图" → 命名为 `Sheet1_Power`、`Sheet2_MCU`…

### 第二步：在子 Sheet 中定义接口

在每个子 Sheet 中，放置 **"引出入口/引出出口"（Port）**——Port 相当于一张 Sheet 的"对外接口"，用于在 Sheet 之间传递信号。如果不放 Port，子 Sheet 内部的信号就没办法暴露给根 Sheet 连接：

- 菜单"放置→端口" 或快捷键
- 每个 Port 起个名字（如 `3V3`、`GND`、`UART_TX`、`RST`）
- Port 会出现在方块图上作为引脚

### 第三步：在根 Sheet 创建方框图

1. 打开根 Sheet（通常命名为 Root）
2. 菜单"放置→电路方框图符号"
3. 双击方框 → 属性 → "连接图纸" → 选择对应的子 Sheet
4. 方框上会自动出现与子 Sheet Port 同名的引脚

### 第四步：连接方框图

在根 Sheet 中用导线或网络标号把各个方框的引脚连起来。

## 📖 什么时候用多 Sheet？

| 场景 | 用多 Sheet？ | 理由 |
|------|-------------|------|
| < 20 个元件，单功能 | ❌ 不用 | 一张图就够了 |
| 20-50 个元件，2-3 个功能模块 | ✅ 推荐 | 按电源/MCU/外设拆分 |
| > 50 个元件，多模块 | ✅ 必须 | 否则根本没法看 |
| 团队协作 | ✅ 必须 | 每人负责一张 Sheet |

## ✏️ 今日作业

把 D5 的 NE555 LED 闪烁电路改成多 Sheet 结构：

1. Sheet1：NE555 定时电路（包含 R1, R2, C1, U1）
2. Sheet2：LED 驱动电路（包含 R3, LED1）
3. 根 Sheet：用方框图把 Sheet1 的输出（OUT）连接到 Sheet2 的输入
4. Sheet2 的 LED 输出标注为 `LED_OUT` 网络

> **完成标志**：根 Sheet 上有两个方框图，导线连接清晰，切换子 Sheet 能看到对应电路。
