# 电路原理图专题 — 开环控制部分

> **学习目标**：理解开环控制的概念，掌握常见开环控制电路的设计方法，学会用定时器/计数器实现无反馈控制。

---

## 一、什么是开环控制？

**开环控制（Open-Loop Control）**：控制器按照预设的指令序列运行，**不检测实际输出结果**，没有反馈信号修正误差。

### 开环 vs 闭环 对比

```
开环：
   输入 ──→ [控制器] ──→ [执行器] ──→ 输出
   没有反馈回路！

闭环：
   输入 ──→ [控制器] ──→ [执行器] ──→ 输出
              ↑                         │
              │       [传感器] ←────────┘
               ←────── 反馈信号
```

### 特点

| 优点 | 缺点 |
|------|------|
| 电路简单，成本低 | 精度低，不能自动纠偏 |
| 响应快（没有采样延迟） | 受元件参数漂移影响大 |
| 稳定无振荡 | 无法适应负载变化 |
| 容易实现 | 需要精确的预校准 |

### 生活中的例子

- **洗衣机定时器**：设定 30 分钟，不管衣服洗没洗干净，到时停
- **流水灯**：NE555 + CD4017 按固定节奏亮灯，不检测灯是否真的亮了
- **步进电机**：发固定脉冲数，假设电机走准了位置
- **电风扇定时**：到时间就停，不检测温度

## 二、常见开环控制电路

### 2.1 RC 定时控制（基础）

最简单的开环控制——用 RC 充放电时间常数控制延时。

```
  VCC ──┬── R(可调/固定) ──┬── 电容正极 ──── 输出信号
        │                   │
        └───────────────────┤
                            C
                            │
                           GND

  延时 T ≈ 1.1 × R × C（NE555 单稳态）
```

### 2.2 NE555 + CD4017 顺序控制（你已经做过的流水灯）

```
                    NE555                    CD4017
  ┌──────────────────────────┐    ┌──────────────────────┐
  │  RP(50k) + 1kΩ + 100μF   │    │  Q0 → 继电器1 (2秒)  │
  │  → 产生时钟脉冲           │───→│  Q1 → 继电器2 (2秒)  │
  │  频率 = 0.693×R×C        │    │  Q2 → 继电器3 (2秒)  │
  │  固定频率，不受负载影响     │    │  ...                 │
  └──────────────────────────┘    │  Q9 → 复位循环       │
                                   └──────────────────────┘
```

这个电路是**纯粹的开环**——NE555 不管 CD4017 是否收到了脉冲， CD4017 不管 LED 是否点亮。

### 2.3 步进电机开环控制

步进电机的转角由脉冲数量决定，是开环控制的经典应用。

```
  MCU ──┬── 步进电机驱动芯片 ──┬── 步进电机
        │  (如 A4988 / DRV8825) │
        │                       │
        ├── STEP（脉冲步进）     │  每来一个脉冲，电机转一步
        ├── DIR（方向）         │  1.8°/步 × 200 脉冲 = 360°
        ├── ENABLE（使能）       │
        └── GND                │  没有反馈——假设电机走准了
                               └──────────────────
```

**为什么不加反馈？**
- 步进电机在低速、低负载下基本不会失步
- 加编码器（闭环）成本增加 3~10 倍
- 应用场景允许丢步的误差（如 3D 打印机的归零）

### 2.4 定时器中断控制（MCU 实现）

MCU 内部的定时器可以做精确的开环时序控制。

```
  ┌─────────────────────────────────────┐
  │  MCU 定时器                          │
  │                                     │
  │  每 1ms：定时器溢出 → 中断            │
  │  累计 1000 次（1秒）→ 切换 LED 状态   │
  │  累计 60000 次（60秒）→ 关闭加热器    │
  │  不需要检测温度，纯粹靠时间推算        │
  └─────────────────────────────────────┘
```

**MCU 开环控制的典型场景：**

| 场景 | 控制方式 | 为什么用开环 |
|------|---------|-------------|
| LED 呼吸灯 | PWM 占空比逐渐变化 | LED 响应快，不需要测亮度 |
| 步进电机走固定距离 | 发固定脉冲数 | 低速不失步 |
| 固件升级指示灯 | 定时闪烁 | 纯粹是信息输出 |
| 定时开关 | RTC 闹钟触发 | 不关心负载状态 |

### 2.5 开环 PWM 控制

PWM（脉宽调制）本身是开环的——设定占空比，假设负载按预期响应。

```
  ┌─────┬─────┬─────┬─────┬─────┬─────┐
  │     │     │     │     │     │     │
  │     │     │     │     │     │     │
  └─────┴─────┴─────┴─────┴─────┴─────┘
  周期 T = 1ms (1kHz)
  占空比 = (高电平时间)/T × 100%

  50%  ──┬──────┐  ┌──────┐  ┌──────
         │      │  │      │  │
         └──────┘  └──────┘  └──────
                 → 电机 50% 速度（假设）
```

## 三、开环电路的优缺点总结

### 什么时候应该用开环？

- ✅ 系统对精度要求不高
- ✅ 负载特性稳定、可预测
- ✅ 成本敏感、空间有限
- ✅ 需要极快的响应速度
- ✅ 系统不可能施加反馈（如远程控制）

### 什么时候必须抛弃开环？

- ❌ 负载变化大（电机堵转时电流飙升，开环不知道）
- ❌ 精度要求高（恒温箱 ±0.1°C，开环做不到）
- ❌ 安全关键（飞控系统失速了必须知道）
- ❌ 长时间运行会漂移（元器件老化改变参数）

---

## 四、开环控制电路设计 checklist

- [ ] 确定时间基准：RC 定时器 / MCU 定时器 / 外部时钟？
- [ ] 元件参数漂移是否在可接受范围内？（电容老化 ±20%/10年）
- [ ] 温度变化是否影响定时精度？（电解电容在低温下容量下降）
- [ ] 是否需要校准？（出厂时微调电位器）
- [ ] 启动状态是否确定？（上电时输出为低还是高？）
- [ ] 是否考虑了最坏情况？（电源波动 ±10% 时的定时误差）
