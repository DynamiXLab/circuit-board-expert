# 项目5 · 单片机最小系统板

> **封装类型**：📐 全贴片（SMD）——LQFP-48主控 + 0603阻容 + SOP-8 USB转串口  
> **难度**：★★★★☆（最难的是焊LQFP-48和USB差分走线）  
> **画板时间**：1周（约5天，每天2-3小时）  
> **核心教学目标**：掌握MCU最小系统的原理图设计，LQFP封装PCB布局，USB差分走线，为巡线小车主板打基础。

---

## 1. 项目简介

前面四个项目所有电路都是"硬件逻辑"——电路的行为由元件和连线决定，不可更改。单片机（MCU）改变了这个范式：**同一块板子，换一段程序就能做完全不同的事**。

"最小系统板"是一块能让单片机跑起来的最简电路板。它只包含MCU正常工作的必要条件，把所有I/O引脚用排针引出来——像一块"裸奔版Arduino"，可以插到面包板上做各种实验。

我们选用 **STM32F103C8T6**——这是全球最广泛使用的入门级ARM Cortex-M3单片机：
- 72MHz主频
- 64KB Flash / 20KB RAM
- 37个GPIO（通用输入输出引脚）
- 2个SPI、2个I2C、3个USART、1个USB、1个CAN
- 12位ADC（10个通道）
- LQFP-48封装（7×7mm，48脚，脚距0.5mm）

它便宜（约5-8元一片）、资料极其丰富、生态成熟。

### 1.1 你将学到什么

- 单片机最小系统的五大组成部分
- LQFP封装的手工焊接（拖焊或热风枪）
- 晶振的负载电容计算和PCB布局
- USB差分走线的90Ω阻抗控制概念
- 电源去耦网络设计
- 把复杂IC的每个引脚都理解清楚

## 2. 认识LQFP封装

### 2.1 LQFP是什么

LQFP（Low-profile Quad Flat Package，薄型四方扁平封装）——芯片是正方形，四个边都有引脚，引脚像海鸥翅膀一样向外弯。

| 参数 | STM32F103C8T6 的值 |
|------|-------------------|
| 引脚数 | 48 |
| 本体尺寸 | 7mm × 7mm |
| 引脚间距 | **0.5mm**（两根引脚中心到中心的距离） |
| 引脚宽度 | ~0.22mm |
| 焊接难度 | ★★★★☆ |

对比前面的SOP-8（1.27mm间距），LQFP-48的0.5mm间距对焊接精度要求明显更高。**但别怕——用拖焊法一样能手工焊好。**

### 2.2 引脚编号

LQFP封装的1脚标记有两种：芯片一角有一个**小圆点（凹坑）**，或者那个角是切角。从1脚开始，**逆时针**绕一圈到48脚。

```
        1脚(带圆点)
          ↓
    ┌─────────────────┐
 12 │                  │ 36
    │   STM32F103C8T6  │
 13 │                  │ 37
    └─────────────────┘
   24                  48
```

## 3. 单片机最小系统的五要素

要让STM32跑起来，至少需要这五个部分：

### 3.1 电源

STM32F103C8T6的工作电压是 **2.0~3.6V**，我们统一用 **3.3V**。

芯片有多个VDD和VSS引脚（VDD=数字电源正极，VSS=数字地）：

| 引脚 | 功能 |
|------|------|
| 9 | VDD_1（数字电源） |
| 24 | VDD_2 |
| 36 | VDD_3 |
| 48 | VDD_4 |
| 8 | VSS_1（数字地） |
| 23 | VSS_2 |
| 35 | VSS_3 |
| 47 | VSS_4 |
| 5 | VDDA（模拟电源——给ADC用） |
| 6 | VSSA（模拟地） |

**每个VDD引脚附近必须放一颗0.1μF（100nF）的陶瓷电容到VSS——这叫"去耦电容"。** 为什么每个都要放？芯片内部不同区域的电路在不同时刻开关，会产生高频电流尖峰。如果只有一个去耦电容，电流从芯片另一头流过来时走线电感和电阻已经把它衰减了——远水解不了近渴。

此外，在靠近芯片的地方放一颗 **10μF** 的钽电容或MLCC作为"储能池"。

### 3.2 时钟（晶振）

STM32内部有一个8MHz的RC振荡器（HSI），但精度不够（±1%），做串口通信时波特率可能偏。所以加一个**外部8MHz石英晶振**——精度±20ppm（0.002%），串口通信稳稳的。

晶振两端各接一个负载电容（通常12~22pF）到GND：

```
        ┌──────────┐
        │  8MHz    │
        │  晶振    │
        └──────────┘
        │        │
   ┌────┴──┐ ┌──┴────┐
   │ C1    │ │ C2    │
   │ 22pF  │ │ 22pF  │
   └───┬───┘ └───┬───┘
       └────┬────┘
           GND
```

**负载电容的取值**：晶振规格书会标注"负载电容CL"。两个外部电容的等效值 = (C1×C2)/(C1+C2) + Cstray（PCB走线的寄生电容，约3~5pF）。如果CL=20pF，Cstray估算4pF，则(C1×C2)/(C1+C2)需为16pF，所以C1=C2=**33pF** 时等效值为16.5pF≈合适。如果CL=12pF，用22pF即可。

简化做法：**用22pF的电容，一般都能起振。**

STM32还有一个32.768kHz的低速晶振（给RTC实时时钟用）——本项目不需要，不焊。

### 3.3 复位电路

NRST引脚（7脚）是低电平复位——平时通过一个 **10kΩ上拉电阻接到3.3V**（保持高电平），需要复位时用一个**按键把NRST拉到GND**。

复位脚对地还并一个 **0.1μF电容**——防止干扰脉冲误触发复位。

```
3.3V
 │
 10kΩ
 │
 ├── NRST(7脚)
 │
 ├── [按键] ── GND
 │
 0.1μF
 │
GND
```

### 3.4 程序下载与调试接口

**SWD（Serial Wire Debug）**——只需要2根线（SWCLK时钟 + SWDIO数据）就能下载程序和调试。比JTAG（需要5~7根线）简洁得多。

用一个 **4脚排针（1×4）** 引出：3.3V、SWDIO、SWCLK、GND。

连接ST-Link/V2调试器（淘宝十几块钱一个）即可烧录和单步调试。

### 3.5 BOOT配置

STM32有两个BOOT引脚决定启动模式：

| BOOT0 | BOOT1 | 启动模式 |
|-------|-------|---------|
| 0 | × | **从Flash启动（正常运行模式）** |
| 1 | 0 | 从系统存储器启动（出厂固化的串口下载程序） |
| 1 | 1 | 从SRAM启动（调试用） |

平时BOOT0接GND（通过10kΩ下拉电阻），BOOT1任意（通常也接GND）。
需要串口下载时，把BOOT0跳到3.3V。

本项目用一个**跳线帽（排针+短路帽）**让BOOT0可以在GND和3.3V之间切换。

### 3.6 附加：USB转串口

为了方便通过USB和电脑通信，加一颗 **CH340N**（USB转TTL串口芯片，SOP-8封装）：
- CH340N的RXD接STM32的PA9（USART1_TX）
- CH340N的TXD接STM32的PA10（USART1_RX）
- CH340N的UD+和UD-通过USB Micro-B或Type-C插座引出

这样一根USB线就能供电+串口通信+下载程序（配合BOOT0切换）。

### 3.7 数据手册阅读实战：STM32F103C8T6

> 本项目是你第一次面对"几十个引脚、几百页数据手册"的大型芯片。回忆一下项目1学过的**数据手册7章节定位法**，现在用同样的方法快速摸清STM32：

**练习：下载STM32F103C8T6的数据手册，找到以下关键信息：**

| 要找的信息 | 在数据手册中的章节 | 答案线索 |
|-----------|------------------|---------|
| 每个引脚的功能 | Pin Definitions（引脚定义表） | 一个脚可能有多个复用功能（如PA9 = USART1_TX / TIM1_CH2），选你需要用的 |
| 电源引脚是哪些 | Pin Definitions | 搜"VDD""VSS""VDDA""VSSA"——STM32有4对VDD/VSS+1对模拟电源 |
| 工作电压范围 | Electrical Characteristics | VDD = 2.0~3.6V → 我们选3.3V |
| 最大电流（单个IO） | Electrical Characteristics | 单个IO最大灌/拉电流约25mA——驱动LED要加限流电阻！ |
| 晶振接哪个脚 | Pin Definitions + Clock Tree | OSC_IN = PD0（5脚），OSC_OUT = PD1（6脚） |
| 启动模式 | Boot Configuration | BOOT0和BOOT1的三种组合——见本项目3.5节 |
| SWD接口引脚 | Pin Definitions | SWDIO = PA13，SWCLK = PA14 |
| LQFP-48封装尺寸 | Package Information | 本体7×7mm，脚距0.5mm——对照5.5节扇出策略 |

> 💡 **拿到一个新MCU的标准动作**：①找引脚定义表 ②找电源引脚和电压范围 ③找最小系统需要的外围（晶振、复位、BOOT）④找下载/调试接口 ⑤找你要用的外设（串口/SPI/I2C/TIM）对应的引脚。这5步做完，你就可以开始画原理图了。

### 4.3 GPIO分配

STM32F103C8T6有37个GPIO，我们要全部引到排针上。其中有些引脚有特殊功能，标注清楚：

| 被占用的引脚 | 功能 |
|-------------|------|
| PA9 | USART1_TX → CH340N RXD |

## 5. PCB布局布线

### 5.1 布局

```
┌──────────────────────────────────────┐
│ [USB] ─ [CH340N]                     │
│                                      │
│   ┌─────────────┐                    │
│   │ STM32F103C8 │  [晶振]            │
│   │   LQFP-48   │                    │
│   └─────────────┘                    │
│                                      │
│ [GPIO排针左列]    [GPIO排针右列]      │
│                                      │
│  [SWD]  [BOOT跳线]  [复位键]  [LED]  │
│           [AMS1117]                  │
└──────────────────────────────────────┘
```

- USB和CH340N在板子左上角，靠近STM32的PA9/PA10
- STM32居中偏上
- 晶振紧贴STM32的OSC_IN(5脚PD0)/OSC_OUT(6脚PD1)
- GPIO排针在STM32两侧（像蝴蝶翅膀展开）
- SWD接口、复位键、BOOT跳线、LED在板子下方

### 5.2 晶振布局（非常重要）

- 晶振**紧贴**MCU的OSC_IN和OSC_OUT引脚——走线越短越好（<10mm）
- 两个负载电容分别靠近晶振两端
- 晶振底下铺GND铜（保护环）
- 晶振周围不要走其他高速信号线

### 5.3 去耦电容布局

- 每个VDD引脚的0.1μF电容**放在MCU背面**（如果元件都在顶层，电容放在底层对应位置）——用最短的路径从VDD经过电容到达VSS
- 做不到放背面的话就放在MCU对应引脚旁边，走线尽量短

### 5.4 USB差分走线（重要）

USB D+和D-是一对**差分信号（Differential Signal）**——信息通过两根线之间的电压差来传递，而不是单根线对地的电压。差分信号抗干扰能力强，因为外部噪声会同时耦合到两根线上，差值不变。

**对于全速USB（12Mbps，CH340N就是这个速率），走线必须遵守以下规则：**

| 规则 | 怎么做 | 为什么 |
|------|--------|--------|
| **并行走线** | D+和D-两根线始终并排走在一起，间距固定 | 保证两根线受到的干扰一致 |
| **等长** | 两根线长度差 ≤ 5mm | 信号到达时间一致，不会畸变 |
| **尽量短** | CH340N的UD+/UD-到USB座 D+/D- ≤ 30mm | 全速USB很容易满足 |
| **不过孔** | D+/D-最好在同一层走完，不要换层 | 过孔引入阻抗不连续、信号反射 |
| **包地** | 差分线两侧各有一条GND走线，或用GND铺铜包围 | 隔离其他信号的串扰 |
| **串联电阻** | 22Ω匹配电阻靠近CH340N一侧放置 | 抑制信号过冲和振铃 |
| **USB座接地** | USB座的金属外壳焊盘接GND | 屏蔽、ESD泄放 |

> 💡 **差分阻抗控制**：严格来说USB差分阻抗应为90Ω。对于双层FR4板，约需线宽0.4mm、间距0.2mm。但全速USB对阻抗容差很宽容（±30%都能工作），不必过度焦虑——遵守上面的规则基本就能通。

**典型走线示意**：

```
CH340N(UD+) ──[22Ω]──┬──────────────┬── USB_D+
                      │  平行等间距   │
CH340N(UD-) ──[22Ω]──┴──────────────┴── USB_D-
                      
两侧和中间用GND铜包围（包地）
```

### 5.5 LQFP-48扇出

LQFP-48的焊盘间距0.5mm，从焊盘拉出线需要技巧：
- 从焊盘拉出0.3mm后立即转45°角
- 内部两排焊盘交替走不同方向
- 利用过孔换层——有些信号走顶层，有些走底层

## 6. 焊接指南

### 6.1 LQFP-48焊接（拖焊法）

这是整个教程里焊接难度最大的一步。但只要方法正确，成功率很高。

**准备工作**：
- 烙铁温度：330~350°C
- 烙铁头：刀头（K型）或马蹄头（BC型）——拖焊的关键
- 助焊剂：**多涂！涂满所有焊盘！**
- 吸锡带：处理连锡

**步骤**：

1. **定位**：把芯片放在焊盘上，对齐（用放大镜或手机微距检查每个引脚是否都在焊盘正中）
2. **固定两个对角**：先焊一个角的引脚固定芯片，再检查对齐，然后焊对角的引脚——这会芯片就固定住了
3. **涂助焊剂**：在一边的12个引脚上涂足助焊剂
4. **拖焊**：烙铁头带少量锡，从引脚根部向引脚尖端方向匀速拖动（2秒左右拖完一边12个脚）——助焊剂会让锡只粘在焊盘和引脚上，不会桥接
5. **检查**：如果两个脚之间连锡了，涂助焊剂再拖一次，或用电烙铁+吸锡带吸掉多余的锡
6. 重复另一边

> 💡 **新手最容易犯的错误**：锡太少（虚焊）或锡太多（连锡）。刚好的焊点是：引脚和焊盘之间的锡形成**微凹的半月形**，光亮饱满。

### 6.2 0603阻容焊接

0603比0805小一圈（1.6×0.8mm），但焊接方法一样——先固定一端再焊另一端。镊子要细、要稳。

## 7. 测试

### 7.1 上电前检查

- 万用表通断档测3.3V和GND不短路
- 万用表通断档测5V和GND不短路
- 目检LQFP-48所有引脚——有无连锡？有无虚焊（引脚没贴在焊盘上）

### 7.2 上电测试

1. 通过USB供电（先不插STM32芯片——如果你用了IC座的话。注意：LQFP-48没有IC座可用，直接焊）
2. 测AMS1117输出是否为3.3V（±0.1V）
3. 断电，接ST-Link到SWD接口
4. ST-Link连电脑，打开STM32CubeProgrammer或ST-Link Utility
5. 点击"Connect"——**成功识别芯片 = 最小系统基本正常**（电源+复位+SWD至少是对的）
6. 烧录Blink程序（PC13的LED闪烁）
7. LED开始闪烁 → **恭喜！你的第一块MCU板子活了！**

### 7.3 串口测试

1. USB连接电脑
2. 烧录一个串口打印程序（`printf("Hello PCB!\n")`）
3. 打开串口助手，选对应COM口，波特率115200
4. 收到 "Hello PCB!" → USB转串口正常

## 8. 常见问题

| 现象 | 可能原因 |
|------|---------|
| ST-Link连不上 | SWD引脚虚焊/连锡/NRST没上拉/电源不对 |
| 连上但烧录失败 | BOOT0没接地（Flash启动需要BOOT0=0） |
| 晶振不起振 | 负载电容不对/晶振虚焊/PCB走线太长 |

## 10. 拓展思考

### 10.1 为什么要用外部晶振而不是内部RC？

内部HSI（8MHz RC振荡器）的精度是±1%（25°C时），全温度范围更差（-2%~+2.5%）。如果用它做串口通信（115200波特率），速率偏差可能导致接收端采样时机偏移，出现误码。外部石英晶振精度±20ppm（0.002%），串口通信稳如磐石。

但如果你对时钟精度要求不高（比如只做LED闪烁、按键读取），完全可以不用外部晶振——省两个电容和一个晶振，引脚还能当GPIO用。

### 10.2 什么是BOD（Brown-Out Detection）？

BOD是STM32内部的"电源电压监控器"——当VDD低于设定阈值（如2.7V）时自动复位芯片，防止在电压不足时发生不可预测的行为（如Flash读写错误）。在STM32CubeMX中配置，默认一般是开启的。

### 10.3 CH340N vs CH340G vs CP2102

- **CH340N**（本项目选用）：SOP-8封装，内置晶振，外围只需两个去耦电容——最简洁
- CH340G：SOP-16封装，需要外接12MHz晶振——引脚多，体积大
- CP2102：QFN-28封装，更小但手工难焊——适合量产
