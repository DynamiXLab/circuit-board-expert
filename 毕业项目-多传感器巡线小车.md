## 1. 项目概述

这是整个教程的毕业设计——设计并制作一辆能够**自动沿着黑线行驶**的智能小车，并且集成多种传感器实现避障、测速、无线调试等扩展功能。

和前面五个项目最大的不同是：这次你不是在"做一块功能板"，而是在"**做一套系统**"——需要同时考虑硬件架构、供电方案、软件算法、机械结构之间的配合。

### 2.3 GPIO分配

STM32F103C8T6 有37个GPIO，需要仔细分配。

| 外设 | 占用引脚 | 说明 |
|------|---------|------|
| 灰度传感器×5 | PA0~PA4 | ADC或GPIO数字输入 |
| 电机A IN1/IN2 | PB12/PB13 | 方向控制 |
| 电机A PWM | PA6 (TIM3_CH1) | 速度控制 |
| 电机B IN3/IN4 | PB14/PB15 | 方向控制 |
| 电机B PWM | PA7 (TIM3_CH2) | 速度控制 |
| 编码器A | PA8/PA9 (TIM1_CH1/CH2) | 左轮测速 |
| 编码器B | PB6/PB7 (TIM4_CH1/CH2) | 右轮测速 |
| 超声波 Trig | PC0 | 触发信号 |
| 超声波 Echo | PC1 | 回波信号（定时器捕获） |
| 蓝牙 USART3 | PB10(TX)/PB11(RX) | 无线通信 |
| OLED I2C | PB8(SCL)/PB9(SDA) | 0.96寸OLED |
| MPU6050 I2C | 共用PB8/PB9 | I2C总线可挂多个设备 |
| 用户按键 | PC13 | 模式切换 |
| 用户LED | PC14 | 状态指示 |
| 蜂鸣器 | PC15 | 报警提示 |
| SWD | PA13/PA14 | 调试 |
| USB串口 | PA10/PA11 | USB转串口(可选，用项目5的CH340N电路) |

> **引脚复用注意事项**：TIM1_CH1/CH2 用了 PA8/PA9，但 PA9 也是 USART1_TX。这里选编码器用TIM1时，USB串口功能只能用USART2或USART3替代。如果用USART1做串口调试（CH340N），则编码器换到其他定时器。

## 4. 硬件设计要点（进阶）

### 4.1 TB6612 电机驱动（SSOP-24）

这是你们第一次接触SSOP封装——24脚，引脚间距0.65mm，比SOP-8密一倍。焊接方法和SOP-8一样（拖焊法），但需要更细的烙铁头。

TB6612比项目2用的L298N先进得多：
- MOSFET输出（L298N是BJT），导通电阻低（0.5Ω），发热小
- 体积小（5×6.4mm vs L298N的大块头）
- 内置续流二极管（不需要外接8个二极管！）
- VM（电机电压）最高15V，本项目用5V（从MP1584来）
- VCC（逻辑电压）2.7~5.5V，接3.3V
- 每路持续电流1.2A，峰值3.2A

### 4.2 编码器测速

光电编码器（如常见的20线码盘）输出A/B两相方波。通过STM32定时器的**编码器模式**直接读取：
- A相上升沿时看B相电平——B高=正转，B低=反转
- 同时计脉冲数，除以每圈脉冲数得到转速

有了速度反馈，可以在巡线PID外面再套一层**速度环**——让小车速度更稳定。

### 4.3 地平面分割

这是巡线小车主板最容易被忽视但最重要的问题。板子上有三种"地"：

| 地区域 | 包含 | 特点 |
|--------|------|------|
| 功率地 | 电池负极、MP1584地、TB6612电机地 | 大电流、可能有噪声 |
| 数字地 | STM32、蓝牙、OLED、按键 | 高频数字开关噪声 |
| 模拟地 | 灰度传感器（如果用ADC） | 对噪声敏感 |

**策略**：在PCB上划分三个地区域，最终在电池GND入口处**单点汇合**（用0Ω电阻或磁珠短接）。这样功率电流不会流经数字地，数字开关噪声不会耦合到模拟地。

如果不分地直接铺一整块铜——大概率也能工作，但ADC读数会跳、蓝牙偶尔断连。分地是"90分到95分"的优化。

## 5. PID巡线算法（软件核心）

### 5.1 问题建模

5个灰度传感器从左到右编号S1~S5。我们需要计算一个**偏差值（Error）**——正=偏右了，负=偏左了。

**加权平均法**（推荐，适合ADC模拟读取）：

```
传感器位置权重：S1=-2, S2=-1, S3=0, S4=+1, S5=+2
各传感器读数归一化到0~1（或直接用ADC值）

Error = Σ(weight_i × value_i) / Σ(value_i)
```

> 例：如果黑线在S2和S3之间，S2读0.8、S3读0.4，其他≈0  
> Error = (-1×0.8 + 0×0.4) / (0.8+0.4) ≈ -0.67 → 偏左，需要右转修正

**二值法**（比较器输出0/1）：
```
找到黑线中心位置 = (最左发现的0 + 最右发现的0) / 2
Error = 实际位置 - 期望位置(S3=2)
```

### 5.2 PID公式

$$Output = K_p \cdot Error + K_i \cdot \int Error \cdot dt + K_d \cdot \frac{d(Error)}{dt}$$

| 项 | 作用 | 直观理解 |
|----|------|---------|
| **P（比例）** | Error越大，修正越猛 | 偏得越多，方向盘打得越狠 |
| **I（积分）** | 累计Error，消除稳态偏差 | 长时间偏左→逐渐加大右转力度→回正 |
| **D（微分）** | Error变化速率，抑制过冲 | 已经在快速回正了→提前减小修正力度，避免冲过头 |

### 5.3 代码框架（C语言伪代码）

```c
float Kp = 2.5, Ki = 0.05, Kd = 1.0;  // 初始值，后续调参
float integral = 0, last_error = 0;
float target_speed = 60;  // 目标速度 (0~100)

while(1) {
    // 1. 读取5路传感器，计算偏差
    float error = calculate_error(sensor_values);
    
    // 2. PID计算
    integral += error;
    float derivative = error - last_error;
    float correction = Kp * error + Ki * integral + Kd * derivative;
    last_error = error;
    
    // 3. 计算左右轮速度
    float left_speed  = target_speed + correction;   // 左轮加速=右转
    float right_speed = target_speed - correction;   // 右轮减速=右转
    
    // 4. 限幅
    left_speed  = constrain(left_speed,  -100, 100);
    right_speed = constrain(right_speed, -100, 100);
    
    // 5. 驱动电机
    motor_left(left_speed);
    motor_right(right_speed);
    
    delay_ms(10);  // 控制周期10ms = 100Hz
}
```

### 5.4 PID调参口诀

1. 先把Ki和Kd设为0，只调Kp——从小加到大，直到小车开始来回振荡
2. 把Kp减到振荡临界值的60~70%
3. 加Ki——消除稳态偏差（小车在直线上偏一边不走正中）
4. 加Kd——抑制转弯时的过冲和振荡
5. 提高目标速度，重新微调

> **经验**：巡线小车通常Kp主导（2~5），Ki很小（0.01~0.1），Kd适中（0.5~2）。不同赛道、不同传感器高度参数都会变化——学会调参比背参数重要。

```
Level 4: 串口打印5路传感器读数 → 手动推车看读数变化
Level 5: 在赛道上跑开环（固定PWM）→ 车能动
Level 6: P控制巡线 → 能跟线但左右摇摆
Level 7: PD控制巡线 → 平稳但可能偏一边
Level 8: PID控制巡线 → 又稳又正中 → 调高速度
```

### 6.2 调试利器：蓝牙串口实时回传

巡线时小车在跑，不可能连着ST-Link看变量。用蓝牙把Error、correction、left_speed、right_speed实时发到手机或电脑上的串口助手——**这是调试PID最有效的方法**。

## 7. 主板vs模块化方案讨论

本项目要求做**一块集成主板**。但你也可以选择**"主板+传感器子板"**：

- **主板**：STM32 + TB6612 + 电源 + 外设接口（做一块板）
- **传感器子板**：5路灰度传感器（复用项目3的设计），通过FPC排线连接主板

这样的好处：
- 传感器高度可调（子板通过铜柱固定，可以上下调节）
- 主板尺寸更小
- 传感器坏了只换子板

## 8. 常见问题

| 现象 | 可能原因 |
|------|---------|
| 小车不走直线 | 两个电机参数不一致——加左右轮速度补偿系数 |
| 巡线时左右振荡越来越大 | Kp太大/Kd太小/控制周期太长 |
| 转弯时脱线 | 速度太快/传感器间距太大覆盖不到黑线/Kp不够 |
| 直线上也偏一边 | 传感器安装不对称/需要调Ki来消除稳态偏差 |
| 电机转速不匹配 | 电压下降（电池没电了）/建议加编码器做速度闭环 |
| 蓝牙断连 | 电机干扰——蓝牙天线远离电机走线/加磁珠/用2.4G模块替代蓝牙 |
| TB6612烧了 | 电机堵转电流过大/VM超过15V/没有散热 |
| ADC读数跳动 | 模拟地没和数字地分开/电机干扰耦合 |

## 10. 写在最后

巡线小车可能是很多同学人生中第一个"完整创造"的电子产品——从系统设计、PCB Layout、焊接、到软件算法——全部自己完成。

它大概率不会一次成功。传感器阈值不对、电池没电了、晶振没起振、电机线焊反了、PID参数完全不对、跑两步就脱线……这些都是正常的过程。调试这些问题的过程，比做出来一辆完美的小车学到的东西多得多。

**记住两句话**：
1. 先跑起来，再跑好——第一版不需要PID控制，P控制能跟着线走就是胜利
2. 示波器和万用表是你最好的朋友——"猜"永远不如"测"
